### Question 2

**By consulting the displayed information in Wireshark's packet content field for this packet (or by consulting the textbook), what is the length (in bytes) of each of the first two fields in the UDP header?**

Each of the four UDP header fields is exactly **2 bytes (16 bits)** long, giving a total UDP header size of **8 bytes**.

| Field | Size |
| --- | --- |
| Source Port | 2 bytes |
| Destination Port | 2 bytes |
| Length | 2 bytes |
| Checksum | 2 bytes |

### Question 3

**What is the value in the Length field of this UDP header? What is the maximum number of payload bytes that can be carried in a UDP segment?**

- **Length field value:** 43 bytes
- The Length field specifies the total size of the UDP datagram — that is, the 8-byte header **plus** the application-layer payload.
- **Payload bytes in this packet:** 43 − 8 = 35 bytes

### Question 4

**What is the maximum number of bytes that can be included in a UDP payload?**

The `Length` field is 16 bits, so its maximum value is **65,535**. Since the UDP header is 8 bytes, the maximum payload is:

$$65,535 - 8 = 65,527 \text{{ bytes}}$$

### Question 5

**What is the largest possible source port number?**

The Source Port field is 16 bits, so the largest possible value is **65,535** ($2^{16} - 1$).

### Question 6

**What is the protocol number for UDP in the IP header? Examine the Protocol field in the IP header of the packet carrying this UDP segment.**

- **IP Protocol field value:** 17
- The protocol number for UDP is **17** (0x11). This tells the IP layer to pass the payload up to the UDP module.

### Question 7

**Examine the pair of UDP packets in which your host sends the first UDP packet and the second UDP packet is a reply. Describe the relationship between the port numbers in the two packets.**

| | Packet # | Source Port | Destination Port |
| --- | --- | --- | --- |
| **Request** | 2 | 63587 | 53 |
| **Response** | 3 | 53 | 63587 |

**Relationship:** The source port of the request (63587) becomes the destination port of the response, and the destination port of the request (53) becomes the source port of the response. This "port swap" is how the reply is routed back to the correct application process on the requesting host.
