# Wireshark Lab 2 — ICMP

## Overview

This lab explores several aspects of the ICMP protocol as described in**Kurose & Ross, *Computer Networking: A Top-Down Approach*, 8th ed., §5.6**:

- ICMP messages generated by the **Ping** program
- ICMP messages generated by the **Traceroute** program
- The format and contents of an ICMP message

`icmp_lab.py` automates the full workflow:

1. Detects the active network interface via `tshark`
2. Starts a background packet capture (`.pcap`)
3. Runs `ping` (10 ICMP Echo Requests)
4. Runs `traceroute`
5. Waits for the capture to finish
6. Parses the `.pcap` with **pyshark**
7. Prints a complete annotated lab report answering all 10 questions

## Usage

### Live capture (most common)

Packet capture requires elevated privileges because it reads raw network frames.

```bash
# Target: HKUST (Hong Kong) — good for demonstrating cross-Pacific latency
sudo python icmp_lab.py www.ust.hk

# Custom ping count and capture duration
sudo python icmp_lab.py www.ust.hk --count 10 --duration 90

# Specify the capture interface manually (skip auto-detection)
sudo python icmp_lab.py www.ust.hk --interface en0

# Full CLI help
python icmp_lab.py --help
```

> **macOS note:** On Apple Silicon and recent macOS versions, `tshark` may
> require that Wireshark itself has been granted the "Local Network" permission
> in _System Settings → Privacy & Security_.

### Offline analysis (no capture needed)

If you already have a Wireshark trace (e.g., the sample traces from
`gaia.cs.umass.edu/wireshark-labs/wireshark-traces.zip`):

```bash
python icmp_lab.py --analyze-only data/ICMP-ethereal-trace-1
```

### CLI reference

| Flag             | Short | Default                  | Description                |
| ---------------- | ----- | ------------------------ | -------------------------- |
| `host`           | —     | `www.ust.hk`             | Target hostname / IP       |
| `--count`        | `-c`  | `10`                     | Ping echo request count    |
| `--output`       | `-o`  | `data/icmp_capture.pcap` | Output `.pcap` path        |
| `--duration`     | `-d`  | `60`                     | Capture duration (seconds) |
| `--interface`    | `-i`  | _(auto)_                 | Network interface name     |
| `--analyze-only` | —     | —                        | Path to existing `.pcap`   |
| `--verbose`      | `-v`  | —                        | Enable DEBUG logging       |

## Security Notes

- The `host` argument is passed to `subprocess.run` as a **list element**, never through a shell string, preventing shell-injection attacks.
- Packet capture is restricted to ICMP traffic via a BPF capture filter; no unrelated traffic is written to disk.
- The `data/` directory (containing `.pcap` files) is excluded from version control via `.gitignore`.
